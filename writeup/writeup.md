# Write-up 보고서 가이드

사용자가 "write-up" 또는 "write-up 보고서 작성"이라고 하면 아래 형식으로 출력한다.

============================================================
## 보고서 형식
============================================================

```markdown
## [문제명] Write-up

### 문제 개요
- 카테고리 / 목표 / 초기 조건 (2~3줄)

### 풀이 과정
- 단계별로 번호를 매겨 서술
- 각 단계: 무엇을 했고, 왜 했고, 결과가 무엇인지
- 핵심 코드/명령어는 코드 블록으로 포함
- 불필요한 시행착오는 생략, 성공 경로만 기술

### 취약점 체인 요약
- 공격 흐름을 한눈에 보이게 화살표(→)로 연결

### Flag
- flag 값 출력
```

============================================================
## 캡처 안내 규칙
============================================================

각 단계에서 캡처가 필요한 지점을 `[📸 캡처]` 태그로 표시한다.

**필수 캡처 대상:**
- 취약점 발견 근거 (취약한 코드, 설정, 응답 등)
- exploit 실행 전후 상태 변화 (balance 변화, 권한 변화 등)
- 핵심 공격 요청과 응답 (Burp Suite, curl 결과 등)
- flag 획득 화면 (최종 결과)

**캡처 안내 형식:**
`[📸 캡처: (무엇을) - (어디서/어떻게)]`

예시:
- `[📸 캡처: race condition 성공 응답 - Burp Suite Intruder 결과 또는 터미널 출력]`
- `[📸 캡처: balance 변화 확인 - /mypage 화면]`
- `[📸 캡처: flag 획득 - PDF 내용 또는 응답 화면]`

============================================================
## 서술 원칙
============================================================

- 간결하게. 한 단계에 3~5문장 이내.
- 왜 이 공격이 가능한지 원인을 1문장으로 설명.
- 실패한 시도는 언급하지 않는다 (문제 이해에 필수적인 경우 제외).
- 방어 방법은 쓰지 않는다 (요청 시 제외).

끝.
